Cloudflare + Backblaze B2 搭建图床

发表于2022-12-21|更新于2022-12-21|编程
|阅读量:

注册Backblaze B2并配置

注册过程非常简单，只需要填写邮箱和密码。注册完后登陆帐号，在 我的设置 里验证一下邮箱

为你的B2图像创建一个存储桶

登录Backblaze，点击左侧B2 Cloud Storage下的Buckets，然后点击Create a Bucket。您需要给您的存储桶一个唯一的名称，并将其公开。保留其他设置的默认值。

最后，单击上传/下载并将测试文件上传到您的新存储桶。

单击该文件可查看其详细信息，包括其各种 URL。

记下友好 URL中的主机名。正如您在上一段中看到的，我的是f004.backblazeb2.com

注册Cloudflare并配置

注册过程非常简单，只需要填写邮箱和密码。注册完后登陆帐号，在 我的设置 里验证一下邮箱

为你的图像主机创建 DNS 子域

在Cloudflare添加需要加速的顶级域名，如：baidu.com

修改腾讯云DNS解析为CloudFlare


点击你创建好的域名进入DNS配置

在DNS配置中配置相关信息，Type选择CNAME，Name输入www 访问路径则为：www.baidu.com，Content为上面记下的f004.backblazeb2.com，通过www.baidu.com指向f004.backblazeb2.com域名

如果您现在通过转到您的自定义子域中的测试文件的 URL 进行测试，例如，https://files.example.com/file/my-unique-bucket-name/smiley.png 几秒钟后，您将看到来自 Cloudflare 的 522“连接超时”错误：

这是因为默认情况下，Cloudflare 通过纯 HTTP 而不是 HTTPS 访问上游服务器。Backblaze 仅支持安全的 HTTPS 连接，因此 HTTP 请求失败。为了解决这个问题，在 Cloudflare 仪表板的 SSL/TLS 部分，将加密模式从“灵活”更改为“完全（严格）”，以便 Cloudflare 通过 HTTPS 连接到 Backblaze，并且需要 CA 颁发的证书。

现在您应该能够通过表单的 URL 访问您的自定义子域中的测试文件 https://www.baidu.com/file/metadaddy-public/smiley.png ，下一个任务是创建/file/metadaddy-public要从 URL 中删除的第一个转换规则。

重写传入请求的 URL 路径

创建一个URL Rewrite，隐藏/file/metadaddy-public路径

创建一个HTTP Response Header Modification，从响应中删除 HTTP 标头和通过 ETag 和 Cache-Control HTTP 标头优化缓存效率

创建一个HTTP Response Header Modification，设置缓存，从而减轻 Cloudflare 和 Backblaze B2 的负担

创建一个HTTP Response Header Modification，为图像文件设置 CORS 标头
